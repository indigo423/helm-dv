# helm-dv

Helm chart for [Delta-V](https://github.com/pbrane/delta-v) — composable,
containerized OpenNMS Horizon on Kubernetes.

The chart deploys the Delta-V application plane (12 Spring Boot daemons, the flow
pipeline, and the Minion ingress plane), with PostgreSQL, Kafka, ClickHouse,
VictoriaMetrics and Grafana available as optional, condition-gated subcharts.
It is derived from Delta-V's `deploy/compose.yml`; Compose stays the local
dev/E2E path, this chart is the Kubernetes surface.

Full chart documentation: [`charts/deltav/README.md`](charts/deltav/README.md).

## Install

### From the OCI registry (GHCR)

```bash
helm install deltav oci://ghcr.io/indigo423/helm-dv/deltav
```

### From the Helm repository (GitHub Pages)

```bash
helm repo add deltav https://indigo423.github.io/helm-dv
helm repo update
helm install deltav deltav/deltav
```

By default backing services are **external** — set `global.kafka.bootstrapServers`,
`global.postgres.host`, etc. For a self-contained demo that bundles them, use the
[`values-demo.yaml`](charts/deltav/values-demo.yaml) preset.

## Development

`make` is the front door; CI calls the same targets.

| Target | What it does |
|---|---|
| `make deps` | Vendor subchart dependencies |
| `make lint` | chart-testing lint |
| `make unittest` | helm-unittest suites |
| `make kubeconform` | Schema-validate rendered manifests |
| `make docs` / `docs-check` | Regenerate / verify chart README (helm-docs) |
| `make kind-test` | Install onto the current kind context + smoke test |
| `make verify` | Everything above (full local gate) |

Edit `charts/deltav/README.md.gotmpl`, never `README.md` — it is generated by
helm-docs and the CI `docs-check` gate fails on drift.

## CI/CD

| Workflow | Trigger | Does |
|---|---|---|
| **CI** (`ci.yaml`) | PRs + pushes touching `charts/**` | ct lint, helm-unittest, kubeconform schema validation, helm-docs drift check, then a real `helm install` on a kind cluster (pulls the public `ghcr.io/pbrane/*` images) with a health smoke test |
| **Release** (`release.yaml`) | push to `main` with a new chart version | chart-releaser publishes a GitHub Release + `index.yaml` to `gh-pages`, and the chart is pushed to GHCR as an OCI artifact |

GitHub Actions are SHA-pinned and kept current by Dependabot.

> **Pages setup (one-time):** after the first release run, enable GitHub Pages
> for the `gh-pages` branch so `https://indigo423.github.io/helm-dv` serves the
> Helm repo index.
